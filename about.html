站点

静态博客托管于 GitHub Pages，无构建步骤：HTML / CSS / JS 直出，文章为 posts/*.md，索引为根目录 posts.json。

用 Obsidian 当「编辑器」

可以把本仓库克隆到本地，用 Obsidian 打开仓库根目录作为一个 Vault。将笔记放进 posts 文件夹、与线上 Markdown 共用即可。建议：

  • 链接与图片路径：浏览器中的文章页是 post.html?slug=xxx，图片请写相对网站根目录的路径，例如 ![](posts/welcome/demo.svg)，这样部署后与本地 Obsidian 预览都能对齐（把附件放在 posts/文章slug/ 下更易管理）。
  • Wiki 链接：[[另一篇]] 不会被当前站点解析，请改用标准 Markdown 链接 [标题](post.html?slug=...) 或链到外部。
  • 在 Obsidian 里改完后，照常 git commit / push 即可更新线上。

数学公式

正文支持 KaTeX，与 Obsidian 常见写法类似：行内 $...$，块级 $$...$$，亦可用 \( \) 与 \[ \]。详见 Markdown 写作小抄。

后续：朋友圈等内容

若要把朋友圈择优同步到博客，可在试验分支维护独立页面或 JSON 数据源，再合并到主分支发布——当前实验功能在 dev/experiment 分支开发。
