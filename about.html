关于

站点

静态博客托管于 GitHub Pages：HTML / CSS / JS 直出；文章为顶层 posts/*.md（勿把正文只放在子目录里，否则无法生成索引与 post.html?slug= 路径）。列表索引为根目录 posts.json，由推送到 main 时的 GitHub Actions 根据各篇 YAML 头自动生成，无需手改。

文章 YAML（与 Obsidian 兼容）

每篇 posts/某标题.md 开头使用 front matter，例如：

---
title: 我的第一篇博客
date: 2026-04-24
categories:
  - 技术
tags:
  - Markdown
  - 博客
pinned: false
excerpt: 可选；不写则从正文首段自动生成摘要
---

正文 Markdown…

categories 取第一项作为站点里的「分类」字段；也可用单行 category: 技术。slug 由文件名决定（不含 .md）。推送 main 后约 1～3 分钟站点与索引会更新。

用 Obsidian 当「编辑器」

可以把本仓库克隆到本地，用 Obsidian 打开仓库根目录作为一个 Vault。将笔记放进 posts 文件夹、与线上 Markdown 共用即可。建议：

  • 链接与图片路径：浏览器中的文章页是 post.html?slug=xxx，图片请写相对网站根目录的路径，例如 ![](posts/welcome/demo.svg)，这样部署后与本地 Obsidian 预览都能对齐（把附件放在 posts/文章slug/ 下更易管理）。
  • Wiki 链接：[[另一篇]] 不会被当前站点解析，请改用标准 Markdown 链接 [标题](post.html?slug=...) 或链到外部。
  • 在 Obsidian 里改完后，照常 git commit / push 即可更新线上。

数学公式

正文支持 KaTeX，与 Obsidian 常见写法类似：行内 $...$，块级 $$...$$，亦可用 \( \) 与 \[ \]。详见 Markdown 写作小抄。
